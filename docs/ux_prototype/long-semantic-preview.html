§2 便签样式 (ME-036)

折角矩形 · 标题(斜体加粗) · 正文 · 颜色按标题分

Constraint:
some restriction on Type-B
Derivation:
orders where order.total >= 1000 and orderedAt within last 30 days
Note:
临时备忘：待与业务方确认字段命名。
还原 SemanticStatements.png 第 3 张：折角（右上角折起）是视觉核心；标题使用斜体加冒号（Constraint:）；正文左对齐、可多行； 便签整体可自适应内容高度，最小尺寸约 180 × 80。
决策点 §2 · 颜色方案：Constraint = 琥珀/橙（警戒意味）、Derivation = 蓝（逻辑/公式）、Note = 中性灰。
建议：三个标题使用同一米色底（#fffbe6，便签纸），仅顶部标题文字颜色区分，避免整张便签色彩过饱和干扰画布。

§4 附着与连接线 (ME-038)

还原样图：Type-B + Constraint Note

便签与 Type-B 之间用 虚线 连接；连接点在 Type-B 边缘（就近侧），便签侧选其顶边/左边靠近的中点。 移动 Type-B 或便签时，虚线端点自动重算；移动 Type-B 时便签本身的坐标不随动（便签保持绝对位置，仅连线跟随）。

创建流程 · 两种路径

  1. 路径 A · 先空便签后附着：左侧选 Long Semantic → 画布空白处点击 → 生成未附着便签（连接线 = 无）→ 拖到 Type / Relation 附近自动吸附；或右侧面板里手动选择附着目标。
  2. 路径 B · 直接附着创建：左侧选 Long Semantic → 点击 Type 或 Relation → 便签默认出现在目标下方 24px 处，连接线立即建立。

交互细节

  • 选中便签后：双击便签正文进入行内编辑；Enter 换行，Esc 退出；失焦自动保存。
  • 标题类型在右侧 Inspector 下拉切换，便签标题颜色与文字实时更新。
  • 删除便签不影响其附着对象；删除 Type / Relation 时，附着在其上的便签改为「未附着」状态（保留在画布上）。
  • 拖拽便签到另一个 Type / Relation 附近时，显示高亮轮廓预览目标；松开后切换附着。
  • 撤销/重做覆盖：创建、删除、改标题、改正文、改附着、移动位置。

§5 数据模型 & Schema

// src/models/diagram.ts
export type LongSemanticHeading = 'constraint' | 'derivation' | 'note';

export interface LongSemanticElement {
  id: ElementId;                       // "note-xxxx"
  type: 'longSemantic';
  heading: LongSemanticHeading;
  body: string;                        // 多行纯文本
  /** 附着目标；undefined = 未附着，自由摆放 */
  attachedTo?: ElementId;              // Type 或 Relation 的 id
  layout: Layout;                      // 绝对坐标 + 尺寸
}

export type DiagramElement =
  | TypeElement | RelationElement | GeneralizationElement
  | LongSemanticElement;
Schema v1.2 新增 longSemantic 元素类型；heading 为枚举；attachedTo 为可选引用；layout 同 Type 的绝对矩形。 连接线是视觉派生，不作为独立元素保存。

§6 决策点汇总

  1. 折角方向：统一右上角折起（与样图一致），不做用户可配置。
  2. 颜色方案：底色 #fffbe6 统一；标题文字按 heading 区分（琥珀 / 蓝 / 灰）。
  3. 编辑方式：双击便签正文进入行内编辑 + 右侧面板 textarea 同步；右侧面板是主编辑入口，行内编辑是快捷方式。
  4. 附着目标删除：便签降级为「未附着」，而非级联删除。
  5. 拖动 Type 时便签联动：不联动（便签保持绝对坐标，只有虚线重算）。这是否符合预期？
  6. 便签初始尺寸：220 × 100，内容超出自动增高，最大高度 400 后出现内部滚动（或强制换行）。
  7. Derivation 只能附于 Relation 吗？PRD 说「派生映射」。建议第一阶段允许 Derivation 附于任意 Relation（不检查 isDerived），只在 hint 中提示；硬校验留到后续 isDerived 规则完善时再加。
