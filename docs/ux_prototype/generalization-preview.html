语义校正（v2，优先级高于 PRD/Schema 原文）：
· Complete（完整划分） = 容器底部 单 水平线（就是容器矩形自身的下边）
· Incomplete（不完整划分） = 容器底部 双 水平线，多出来那根画在 容器内侧
· 默认创建为 Complete
与当前 prd.md / diagram.schema.json / stories.csv ME-031 描述相反，这三处需在实施时一并订正。

A. 目标视觉语法（修订版）

修订Complete — 单底线（默认）

只有容器矩形自身那条底边 = 完整划分。实例必须落在某个子类里。

修订Incomplete — 双底线（内侧）

多出的一根线画在容器 里面，紧贴底边上方 5px。

视觉Multi-level — 嵌套容器

外层 Party 是 Complete（单底线）；内层 Person 是 Incomplete（双底线，内侧补线）。可任意层嵌套。

视觉同父多次划分 — 容器间不重叠

每个容器独立的 completeness。容器间几何上不得重叠，拖动时若发生碰撞由系统阻止或回退。

视觉水平排布 — 父类在左，容器在右

连接线从父类最近一边接到容器最近一边，自动选接入侧；保持水平/垂直一根线。

视觉水平排布下的 Incomplete 标识

Incomplete 标识永远贴容器底边内侧（不随父类方向旋转），保持 Fowler 惯例。

B. 交互结论（v1 已确认）

  • ME-028：左侧工具栏新增泛化工具按钮，tooltip "创建类型泛化划分容器"，点击后 ToolMode = 'generalization'。
  • ME-029：① 选父 Type（顶部/光标旁显示提示条 "请先选择父 Type，然后在画布空白处点击以放置容器"）→ ② 画布空白单击，在该点落一个固定尺寸空容器（约 220×140），连接线自动从父类最近一侧接入。
  • ME-030：容器内 Type 工具点击 = 新建子 Type；外部 Type 可拖入；子 Type 可拖出变回自由 Type；容器随内容自动收缩/扩展，无手动 resize。
  • ME-031：Inspector 单选 Incomplete / Complete，默认 Complete；视觉实时刷新单线/双线（内侧）。

C. 决策记录（Part C 12 问答案，锁定为实施依据）

#问题决策
1容器创建方式单击落一个固定尺寸空容器（约 220×140）。不做拖拽定尺寸。
2父 Type 绑定时机先选父 Type 再点画布。未选时显示提示条引导。
3连接线语义作为 generalization 元素的渲染细节，不进 relations[]。不可单独选中。
4无父 Type 容器禁止。未选父 Type 时工具按钮点击无效（配合提示条）。
5子 Type 归属容器内 Type 工具新建；或外部已有 Type 拖入容器成为子 Type。
6多层嵌套必须支持，子 Type 可再作为下一层 generalization 的父类。
7同父多次划分必须支持；容器间几何不重叠，碰撞时阻止移动或回退。
8容器大小完全自动（子 bbox + padding），不提供手动 resize。
9拖出容器允许。拖出后自动释放为自由 Type，容器自动收缩。
10删除语义删容器 → 子 Type 全部释放为自由 Type；删父 Type → 级联删除其直接关联的 generalization 容器（等同删容器）。
11默认划分类型Complete（对应单底线）。
12沿用既有设计？结构沿用 Schema generalizationElement { parentTypeId, childTypeIds, completeness, layout } 与 ARCH_DESIGN 的 GeneralizationBox.tsx / ToolMode: 'generalization'。语义字段（complete/incomplete 视觉、默认值）需同步订正。
